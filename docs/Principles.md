# Principles

How Production Hybrid reference modules are built.

1. **Production over cleverness** — Boring, working code beats elegant abstractions.
2. **Small examples over giant demos** — Module Zero does one thing completely.
3. **Real bugs over hypothetical ones** — Every lesson came from shipping a commercial app.
4. **Explain why, not only how** — Postmortems include false leads and impact.
5. **Every recipe must be reproducible** — If it doesn't run locally, it doesn't ship.
6. **Version modules, not just commits** — `v1.0 Production Push` means production-ready.
7. **Solve searches, not showcases** — If nobody is Googling for it, it's not a module yet.
